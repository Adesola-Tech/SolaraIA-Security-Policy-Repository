# Account Password Procedure

## Purpose
To provide steps for creating, issuing, resetting, and protecting passwords within SolaraIA systems.

## Procedure

### 1. Initial Password Assignment
- Assign temporary passwords securely
- Require password change upon first login where supported

### 2. Password Creation
- Ensure password meets minimum complexity requirements
- Encourage use of passphrases where possible

### 3. MFA Enrollment
- Enroll user in MFA for required systems
- Validate MFA activation before granting elevated or remote access

### 4. Password Reset
- Verify user identity before resetting password
- Issue reset instructions through approved channels only

### 5. Compromised Credentials
If a password is suspected to be compromised:
- Force immediate reset
- Investigate related account activity
- Escalate if suspicious activity is detected

## Evidence Retention
Support tickets, approval logs, and MFA enrollment confirmations should be retained according to recordkeeping requirements.
